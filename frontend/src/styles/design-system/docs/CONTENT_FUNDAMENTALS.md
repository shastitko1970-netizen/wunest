# Content Fundamentals

Правила копирайтинга и тона WuNest. Цель: один голос во всём продукте, предсказуемый для пользователя и переводимый без потерь между RU/EN.

---

## 1. Тон

- **Спокойный, компетентный, чуть формальный.** WuNest — инструмент для творческой работы с AI. Юзеру не нужно, чтобы приложение «шутило» или извинялось.
- **Никогда не fake-enthusiasm.** «Awesome!», «Great job!», восклицательные знаки в UI — нет. Восклицательный знак уместен только в error-баннере при реально критичной ошибке.
- **Лаконичность > полноты.** Кнопка — 1 слово. Tooltip — 1 предложение. Empty state — 2 строки + CTA.

## 2. Обращение

- **RU: «вы» без капитализации.** «Войдите, чтобы начать». Без «ты» — это инструмент, не друг.
- **EN: неформальный imperative.** «Sign in to continue.»
- Не обращайся к юзеру по имени в системных строках. По имени — только в чате, где это его персона.

## 3. Casing

- **RU заголовки:** sentence case. «Настройки внешнего вида», не «Настройки Внешнего Вида».
- **EN заголовки:** sentence case тоже. «Appearance settings», не «Appearance Settings».
- Eyebrow/caption в монопространстве — **UPPERCASE with tracking 0.12em**. Только для категорий/секций, не для обычного текста.
- Имена персонажей и названия тем — всегда как автор написал, не трогаем.

## 4. Пунктуация

- Точки в конце пунктов меню, кнопок, chip-ов — **не ставим**.
- Точки в конце предложений в подписях, tooltip'ах, error-сообщениях — **ставим**.
- Em-dash `—` в RU с пробелами: «WuNest — инструмент...». EN: `—` без пробелов в прозе, с пробелами в tech-списках (наш случай).
- Кавычки: RU «ёлочки», EN "typographic" если доступны в шрифте. `'` — только для кода.

## 5. Эмодзи

- **В шелле — никогда.** Навигация, кнопки, сообщения системы, пустые состояния, ошибки — только `mdi-*` icon'ы.
- **В пользовательском контенте — что угодно.** Имена персонажей, сообщения, описания — MD rendering пропускает emoji как есть.
- Unicode-символы как decorative chars (▲, ▍, ·) — ограниченно: логотип `▲`, курсор стриминга `▍`, разделитель в meta-информации `·`. Всё. Никаких ✨, 🔥, 🎉.

## 6. Цифры

- Числа — моноширинный шрифт (`.nest-mono`). Табличные цифры (`font-feature-settings: 'tnum' 1`). Баланс, количество токенов, timestamps — всегда mono.
- Единицы измерения: `px`, `ms`, `KB`, `tok` — строчные, без пробела после цифры в ограниченном UI (chip-ы), с пробелом в проселяном тексте.
  - Chip: `1024tok`.
  - Parag: «Прочитано 1024 токена».

## 7. Примеры из продукта

### Пустые состояния
- «Нет персонажей. Создайте первого, чтобы начать разговор.»
- «Чат пуст. Напишите что-нибудь.»

Два предложения максимум. Никаких «Ой, как пусто!», «Здесь пока ничего нет 😢».

### Баннеры

- Beta gate: «Активируйте код, чтобы разблокировать WuNest. → К аккаунту»
- Safe mode: «Режим безопасности: пользовательский CSS отключён. [Очистить CSS] [Выйти]»

### Tooltip'ы

- «Предыдущий вариант ответа» (не «Swipe left»)
- «Новый вариант» (не «Generate a new swipe!»)
- «Скрыть из промпта» (не «This will hide the message from the AI context»)

### Ошибки

- «Не удалось сохранить. Проверьте соединение и попробуйте ещё раз.»
- Не: «Oops! Something went wrong :(»

### CTA-кнопки

- «Войти», «Создать персонажа», «Сохранить», «Отправить», «Применить».
- Не: «Let's go!», «Start your journey», «Продолжим!».

## 8. RU ⇄ EN паритет

Каждая строка добавляется в `i18n.ts` одновременно на обоих языках. Если EN нет — добавь английский перевод, даже черновой. Нельзя смешивать: половина компонента на RU, половина на EN — блокер для PR.

## 9. Запрещённые формулировки

- «Просто», «всего лишь» — снисходительно. «Войдите всего за секунду!» — нет.
- «AI», «нейросеть», «интеллект» как синонимы — говорим «модель».
- «Бот» — никогда. Есть «персонаж».
- «Токены» — ок, это термин. «Tokens» для EN.
- «Кейс», «фичи», «абьюз» — профессиональный сленг в UI недопустим. В доке — ок.

## 10. Вайб

Представь: **editorial-tech журнал**. Статья про AI в The Verge, но без кликбейта. Функциональный, спокойный, чуть интеллектуальный.
