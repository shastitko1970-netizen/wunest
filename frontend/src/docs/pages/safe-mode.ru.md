# Безопасный режим

Если свой CSS сломал интерфейс настолько, что ты не можешь до настроек добраться — есть спасательный люк.

## Как включить

В адресной строке добавь `?safe` к host'у:

```
https://nest.wusphere.ru/?safe
```

Reload страницы. Custom CSS автоматически отключается **для этой сессии** (не удаляется, а подавляется — так что твои правила остаются).

Сверху появится жёлтый баннер с двумя кнопками:

- **«Очистить CSS»** — удаляет сохранённый custom CSS с сервера. После этого можно снять `?safe`.
- **«Выйти»** — снимает `?safe` из URL и перезагружает. CSS вернётся.

## Что ещё отключается в safe mode

- Custom CSS не инжектится в `<style>`
- Фоновая картинка не ставится на body (unreachable/oversized image может быть причиной зависания)

## Что НЕ отключается

- ST-совместимые переменные из UI (акцент, цвет фона) — применяются как обычно
- Системная тема (светлая/тёмная)
- Размеры шрифта, блюр, тени — все inline-style переменные

## Баннер защищён от overrides

Safe mode баннер — `position: fixed`, `z-index: 100000`, inline `!important` на styles. Даже враждебный CSS не может его спрятать. Плюс — в safe mode кастом CSS и так подавлен, так что это double-защита.
