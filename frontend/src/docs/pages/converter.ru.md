# Конвертер ST → WuNest

`/converter` — LLM-помощник, который берёт твою тему из SillyTavern и переписывает её в WuNest-нативный формат: ST-селекторы (`.mes`, `.mes_block`, `#chat`) на наши классы (`.nest-msg`, `.nest-msg-body`, `.nest-chat-scroll`), удаляет правила на ST-only контейнеры (`.drawer-content`, `#expression-image`), нормализует переменные.

## Когда нужен

- Импортнул `.json` от ST → темa выглядит криво (например, аватары не круглые, swipe-кнопки в неожиданных местах)
- Стащил с Discord/CHUB красивый `.css` от ST-юзера → хочешь применить как WuNest-тему
- Сам пишешь тему и хочешь автоматически рерайтнуть селекторы перед публикацией

Если тема **уже использует** `.nest-*` классы или `--SmartTheme*` переменные — конвертер не нужен, импорт через **Внешний вид → Импорт** работает напрямую.

## Что на входе

Два режима — taby сверху страницы:

### Файл
`.json` или `.css` файл, drag-and-drop или клик. Максимум 500 KB.

- **`.json`** — полный экспорт ST-темы (`{name, main_text_color, ..., custom_css}`). Отправляется LLM как есть.
- **`.css`** — сырой CSS без JSON-обёртки. Авто-оборачивается фронтом в `{name: <имя файла без расширения>, custom_css: <содержимое>}` перед отправкой.

### Вставить текст
Textarea с monospaced шрифтом. Авто-определение:
- Первый non-whitespace символ `{` → JSON. `JSON.parse` валидируется на фронте — malformed JSON блокирует submit (экономит 30-180s ожидания LLM на garbage)
- Иначе → CSS. Авто-оборачивается в `{name: 'Untitled CSS', custom_css: <текст>}`

500 KB лимит ровно как у файла.

## Выбор модели и источника

### Источник
**WuApi pool** или **BYOK** (твой ключ). Для слабых тем (50-200 строк CSS) хватит дешёвой модели. Для бойлерплейта 1000+ строк лучше Claude Sonnet / GPT-4o / Gemini Pro — они меньше промахиваются на edge case'ах.

### Модель
Picker подгружает каталог в зависимости от source'а:
- WuApi pool → все модели тарифа (Sonnet 4.5, GPT-4o, Wu-named тюны)
- BYOK → каталог провайдера твоего ключа (OpenAI / Anthropic / etc.)

## Конвертация

Кнопка **Конвертировать** запускает stream'инг к выбранной модели. **Tokens списываются с твоего ключа** (или wu-gold для WuApi pool):
- Вход: ~system prompt 5K токенов + твоя тема
- Выход: переписанный JSON, обычно эквивалентного размера

LLM-вызов сделан **detached context** на сервере — даже если закроешь вкладку, конвертация добежит до конца и результат будет лежать 24h.

## Лимиты

- **3 конвертации в час** на пользователя (включая retry'и и errored job'ы — ошибочные считаются чтобы не было spam'а)
- 500 KB max input
- 180s timeout на LLM-вызов (жесткий)

При rate-limit'е — 429, в UI таймер до следующей попытки.

## Превью результата

После успеха — блок «Что внутри»:
- **Селекторов переписано в WuNest-стиль: N** — счётчик `.nest-*` упоминаний в выходном `custom_css`. 0 = подозрительно (модель скорее всего эхо-выдала вход обратно)
- **Первые 30 строк CSS** — visual snippet чтобы оценить руками. `<pre>` с monospaced шрифтом, max-height 280px со scroll'ом
- **Заметки** (`_converter_notes`) — что модель сделала: «Заменено `.mes` на `.nest-msg` в 12 местах», «Удалено правило на `.drawer-content` (ST-only)»

## Действия

- **Скачать .json** — download'ит результат как `wunest-theme-<hash>.json`. Поделиться, занести в Git, скинуть другу.
- **Применить себе** — мерджит результат в твой `Appearance.customCss` через тот же путь что и ручной import (`fromST`). Перед apply runs runtime-guard `isPlausibleSTTheme` — если результат не похож на валидную ST-тему (типы полей не совпадают), блокирует apply с warning, оставляя возможность download.
- **Попробовать другую модель** — открывает picker dialog с current model + source pre-selected. Confirm → POST `/api/convert/{id}/retry` с тем же input'ом + новой моделью. Засчитывается в hourly limit.

## История последних конвертаций

Полоса под результатом — последние 24h job'ов. На каждой row:
- Иконка статуса (зелёная — done, красная — error, жёлтая — running)
- Модель + age (`5m ago` / `2h ago`)
- Кнопка download (только для done)
- Кнопка retry (для done или error) — открывает тот же model picker

Через 24h job auto-удаляется reaper'ом, ссылка перестаёт работать. Если нужен permanent — download `.json` пока живой.

## Как написать «хороший» ST-input

Если конвертер ломается на твоём вводе — обычно дело в одном из:

1. **Слишком много `body` правил** — agressive `body { background: ...; color: ... }` в ST-теме. Модель растеряется, как переписывать. Лучше через `:root { --SmartThemeBlurTintColor: ... }`.
2. **JS-only функционал** — ST-плагины через `window.extension_settings.*`. У нас этого API нет, переписать невозможно. Удаляем.
3. **Очень длинная тема (>1000 строк)** — даже сильные модели начинают пропускать в середине. Резай на куски, прогоняй каждый отдельно.
