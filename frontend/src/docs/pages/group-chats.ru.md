# Групповые чаты

WuNest поддерживает групповые чаты — несколько персонажей в одном диалоге. Полезно для сцен с парами/тройками персонажей, где нужно чтобы каждый говорил «своим» голосом, а не один character играл всех ролей.

## Создание

**Библиотека → кнопка «Групповой чат»** открывает GroupChatSetupDialog:

1. Выбираешь 2+ персонажа (мультиselect)
2. Опциональное имя чата (по умолчанию «Group: Alice, Bob, Cyril»)
3. Стиль ротации:
   - **Round-robin** — каждый по очереди
   - **Manual address** — модель отвечает только на @-обращение к конкретному персонажу
   - **Auto** — модель решает сама на основе контекста (требует более сильную модель)

После создания — стандартный chat-view, но с дополнительными свойствами.

## Что меняется в чате

### Несколько `assistant` сообщений подряд
Каждый turn модель отвечает за **одного** персонажа из списка. Persona attribution хранится в `Message.character_id` — UI показывает avatar+имя того кто говорит.

### Swipes per character
Когда регенеришь сообщение, swipes тоже привязаны к конкретному персонажу — `swipe_character_ids` параллельно `swipes`. То есть если у Alice 3 alternative reply'а, переключение swipe не меняет персонажа.

### Адресация
Macro `{{char}}` в group chat'е раскрывается **зависимо от того кто отвечает** в данный turn. Если в карточке Alice написано «{{char}} замечает {{user}} в углу» — это раскрывается с именем Alice.

### Per-character lorebook
Каждый персонаж в группе тащит свой лорбук (если он у него прикреплён). Все они активируются в общем prompt'е — порядок по InsertionOrder, как в одиночных чатах.

## Лимиты и подводные камни

- **Стоимость растёт линейно** с количеством участников — больше системного контекста = больше токенов на каждом ходу
- **Stream'инг быстрый** только для одного persona в turn'е — не пытайся делать «все ответят одновременно», модель в лучшем случае выберет одного, в худшем смешает голоса
- **Контекст-window** — несколько карточек + несколько лорбуков легко съедают 10-20K токенов на одном лишь системном prompt'е. Для длинных групповых сцен нужен модель с 128K+ window

## Удаление участника

Drawer **Настройки чата → Tags** показывает список участников. Можно удалить — `character_ids` обновляется в БД. Но **прошлые сообщения остаются** — character_id в них ссылается на удалённого участника. UI отрендерит их с placeholder-аватаром.
